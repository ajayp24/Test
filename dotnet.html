ASP.NET Web Forms: Page Life Cycle

The ASP.NET Web Forms page life cycle is the sequence of events that occurs when a page is requested and processed. It is important because control values, events, and rendering happen in a fixed order.

  1. Page Request: ASP.NET receives the request and loads or compiles the page.
  2. Start: Page properties are initialized and ASP.NET checks whether the page is a postback.
  3. Init: Controls are initialized and unique IDs are assigned.
  4. Load View State: ViewState values are restored.
  5. Load PostBack Data: Form data is loaded into controls.
  6. Load: Page and controls are fully available.
  7. PostBack Event Handling: Button click and other events are raised.
  8. PreRender: Last chance to modify the page before output.
  9. Save View State: ViewState is saved for the next request.
  10. Render: HTML is generated and sent to the browser.
  11. Unload: Cleanup happens.
if (!IsPostBack)
{
    // Load data only once
}
Easy order: Request → Init → Load ViewState → Load → Events → PreRender → Render → Unload

ASP.NET Web Forms: State Management

HTTP is stateless, so ASP.NET Web Forms provides multiple ways to preserve data across requests.

Client-side State Management

  • ViewState: Stores page and control values in a hidden field.
  • ControlState: Similar to ViewState but cannot be disabled.
  • Hidden Fields: Store small values inside the page.
  • Cookies: Store data in the browser.
  • Query String: Passes data in the URL.
ViewState["Name"] = "Ajay";
string name = ViewState["Name"].ToString();

Server-side State Management

  • Session: Per-user server-side storage.
  • Application: Shared across all users.
  • Cache: Temporary data to improve performance.
  • Database: Persistent storage.
Session["UserName"] = "Ajay";
string name = Session["UserName"].ToString();

ADO.NET

ADO.NET is a .NET data access technology used to connect to databases, execute commands, and retrieve or update data.

Core Components

  • Connection: Opens connection to the database.
  • Command: Executes SQL statements or stored procedures.
  • DataReader: Connected, fast, forward-only data access.
  • DataAdapter: Bridges database and DataSet.
  • DataSet/DataTable: In-memory representation of data.
using (SqlConnection conn = new SqlConnection("connection_string"))
{
    conn.Open();
    SqlCommand cmd = new SqlCommand("INSERT INTO Employees(Name) VALUES(@Name)", conn);
    cmd.Parameters.AddWithValue("@Name", "Ajay");
    cmd.ExecuteNonQuery();
}

Execute Methods

MethodPurpose
ExecuteReader()Reads multiple rows
ExecuteScalar()Returns single value
ExecuteNonQuery()Insert, update, delete

WCF Basics

WCF stands for Windows Communication Foundation. It is used to build service-oriented applications where different systems communicate over protocols like HTTP, TCP, and MSMQ.

ABC of WCF

  • Address: Where the service is located
  • Binding: How to communicate
  • Contract: What the service exposes
Memory trick: Address = Where, Binding = How, Contract = What

WCF Contracts

  • Service Contract: Defines the overall service and the operations it exposes to clients.
  • Operation Contract: Specifies the methods that are available for client calls within the service.
  • Data Contract: Defines the structure of the data exchanged between the service and clients.
  • Message Contract: Provides full control over the SOAP message format, including headers and body.
[ServiceContract]
public interface ICalculatorService
{
    [OperationContract]
    int Add(int a, int b);
}

WCF Examples

Service Contract and Operation Contract

[ServiceContract]
public interface ICalculatorService
{
    [OperationContract]
    int Add(int a, int b);

    [OperationContract]
    int Multiply(int a, int b);
}

Data Contract

[DataContract]
public class Employee
{
    [DataMember]
    public int Id { get; set; }

    [DataMember]
    public string Name { get; set; }
}

Message Contract

[MessageContract]
public class EmployeeRequest
{
    [MessageHeader]
    public string AuthToken;

    [MessageBodyMember]
    public int EmployeeId;
}

How WCF and Web Forms Communicate

In a Web Forms and WCF setup, the Web Forms application acts as the client and the WCF service acts as the service provider.

Flow

  1. User performs action on Web Forms page.
  2. Code-behind calls WCF service using generated proxy.
  3. WCF service processes request.
  4. Response is returned to Web Forms app.
  5. Web Forms displays the result.
protected void btnAdd_Click(object sender, EventArgs e)
{
    CalculatorServiceClient client = new CalculatorServiceClient();
    int result = client.Add(10, 20);
    lblResult.Text = result.ToString();
    client.Close();
}

How Web Forms Knows About WCF

You add a Service Reference in the Web Forms project. Visual Studio then generates the proxy class and updates web.config.

<system.serviceModel>
  <client>
    <endpoint 
      address="http://localhost:1234/CalculatorService.svc"
      binding="basicHttpBinding"
      contract="ServiceReference1.ICalculatorService" />
  </client>
</system.serviceModel>

ASP.NET MVC Overview

MVC stands for Model-View-Controller. It separates application concerns and improves maintainability, testability, and structure.

  • Model: Data and business logic
  • View: UI
  • Controller: Handles requests and coordinates model and view
public class EmployeeController : Controller
{
    public ActionResult Index()
    {
        Employee emp = new Employee { Id = 1, Name = "Ajay" };
        return View(emp);
    }
}

MVC State Management

ASP.NET MVC does not use ViewState. Instead, it provides multiple options to preserve data across requests.

public ActionResult Save()
{
    TempData["Success"] = "Record saved successfully!";
    return RedirectToAction("Index");
}
Best practice: Use strongly typed models for main data instead of ViewBag or ViewData.

MVC Validation

Validation ensures the user enters correct data before saving it. MVC supports both client-side and server-side validation.

public class Employee
{
    [Required(ErrorMessage = "Name is required")]
    [StringLength(50)]
    public string Name { get; set; }

    [EmailAddress(ErrorMessage = "Invalid email format")]
    public string Email { get; set; }

    [Range(18, 60)]
    public int Age { get; set; }
}
[HttpPost]
public ActionResult Create(Employee model)
{
    if (ModelState.IsValid)
    {
        return RedirectToAction("Success");
    }

    return View(model);
}

Validation messages are displayed in the view using helper methods like ValidationMessageFor and ValidationSummary.

What is ActionResult?

ActionResult is the base class for all types of responses that a controller action can return. It provides a common return type for views, JSON, files, redirects, and status codes.

public ActionResult Index()
{
    return View();
}
public ActionResult GetData(bool isJson)
{
    if (isJson)
        return Json(new { Name = "Ajay" }, JsonRequestBehavior.AllowGet);

    return View();
}

Filters in MVC

Filters run code before or after different stages of the MVC request pipeline. They are used for cross-cutting concerns like authorization, logging, exception handling, and caching.

Types of Filters

  • Authorization Filters
  • Action Filters
  • Result Filters
  • Exception Filters
public class MyActionFilter : ActionFilterAttribute
{
    public override void OnActionExecuting(ActionExecutingContext filterContext)
    {
        // Before action executes
    }

    public override void OnActionExecuted(ActionExecutedContext filterContext)
    {
        // After action executes
    }
}

Difference Between Filters and Middleware

FeatureMiddlewareFilters
Runs whereASP.NET Core request pipelineMVC pipeline
Applies toAll requestsMVC actions only
Access to action/model stateNoYes
Common useGlobal logging, auth, exceptionsValidation, action-level logic
Rule: Global request-level concern = Middleware. Controller/action-specific concern = Filter.

What is Short-Circuit?

Short-circuiting means stopping further request processing in the pipeline and returning a response immediately.

app.Use(async (context, next) =>
{
    if (!context.Request.Headers.ContainsKey("AuthToken"))
    {
        context.Response.StatusCode = 401;
        await context.Response.WriteAsync("Unauthorized");
        return; // short-circuit
    }

    await next();
});

If next() is not called, the request does not continue to the next middleware or controller.

Middleware Deep Dive

Middleware is a component in ASP.NET Core that handles HTTP requests and responses in a pipeline.

Basic Structure

public async Task Invoke(HttpContext context)
{
    // Before logic
    await _next(context);
    // After logic
}

Registration

Middleware is registered in Program.cs.

var app = builder.Build();

app.Use(async (context, next) =>
{
    Console.WriteLine("Middleware 1 - Before");
    await next();
    Console.WriteLine("Middleware 1 - After");
});

app.MapGet("/", () => "Hello World!");

app.Run();

Use vs Run vs Map

  • Use: Adds middleware and usually calls next()
  • Run: Terminal middleware, ends pipeline
  • Map: Branches pipeline based on path

Common Built-in Middleware

app.UseExceptionHandler("/error");
app.UseHttpsRedirection();
app.UseRouting();
app.UseAuthentication();
app.UseAuthorization();
app.MapControllers();

What is URL Routing?

URL routing maps an incoming URL to a specific controller and action method.

routes.MapRoute(
    name: "Default",
    url: "{controller}/{action}/{id}",
    defaults: new { controller = "Home", action = "Index", id = UrlParameter.Optional }
);

Example URL:

/Product/Details/10

This maps to ProductController, Details, and id = 10.

ASP.NET Core Web API

ASP.NET Core Web API is used to build RESTful services that return data, usually JSON, for web, mobile, and other client applications.

[ApiController]

public class EmployeeController : ControllerBase

Dependency Injection

Dependency Injection is a design pattern where dependencies are provided to a class instead of creating them inside the class. This improves testability, flexibility, and maintainability.

Without DI

public class EmployeeController
{
    private EmployeeService _service = new EmployeeService();
}

With DI

public class EmployeeController
{
    private readonly IEmployeeService _service;

    public EmployeeController(IEmployeeService service)
    {
        _service = service;
    }
}

Registration in Program.cs

builder.Services.AddScoped<IEmployeeService, EmployeeService>();

Service Lifetimes

LifetimeMeaning
TransientNew instance every time
ScopedOne instance per request
SingletonSingle instance for the whole application
Common interview point: DbContext should usually be registered as Scoped.
